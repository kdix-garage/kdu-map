# What we do?

KDU.map は、近畿大学東大阪キャンパスの屋内外の移動や施設探索を支援するためのマッピングプロジェクトです。

近畿大学東大阪キャンパスは広く、建物の数も多く、建物内部の構造も複雑です。KDU.map は、そのような環境でも利用者が目的地を見つけやすくし、必要な情報に迷わずたどり着ける状態を目指します。

KDU.map は近畿大学非公認プログラミングサークル「KDIX.Garage」によって設計・開発された有志のプロジェクトであり、近畿大学情報学部の「KDIX.Lab」の採択プロジェクトです。

## 目的

KDU.map の目的は、近畿大学東大阪キャンパスにおける「場所が分からない」「建物内部で迷う」「施設情報にたどり着きにくい」といった課題を、地図と検索、経路案内、施設情報の提供によって解決することです。

単にキャンパスの地図を表示するだけではなく、建物・教室・施設・イベント会場などの情報を扱える、実用的なインドア WebGIS アプリケーションとして開発します。

最初のリリースでは、2026年11月に開催予定の生駒祭に向けて、イベント来場者が会場や企画の場所を地図上で確認できるビューワーの提供を目指します。

## 対象ユーザー

KDU.map は、主に以下の利用者を想定しています。

- 新入生・在学生
- 教職員
- 来客者
- 受験生やオープンキャンパス参加者
- 文化祭などのイベント来場者
- 障害のある方や、バリアフリー経路を必要とする方
- 将来的には、施設管理者や地図データの編集者

ただし、最初のリリースでは文化祭などのイベント参加者を主な対象とし、段階的に在学生、来客者、バリアフリー経路を必要とする利用者へ対象を広げます。

## 解決したい課題

近畿大学東大阪キャンパスでは、以下のような課題があります。

- 建物名や号館名だけでは、実際の場所を直感的に理解しにくい
- `23-XXX` のような教室番号を見ても、どの建物のどこにあるのか分かりにくい
- アカデミックシアター（ACT）のような複雑な建物では、目的の書架や施設にたどり着きにくい
- 建物内部の階層構造や出入口、階段、エレベーターの位置が分かりにくい
- バリアフリー経路や車椅子で移動しやすいルートを事前に把握しにくい
- 文化祭などのイベント時に、一時的な会場情報や企画情報を地図上で扱いにくい
- 自動販売機、トイレ、休憩場所などの日常的に使う施設の場所が分かりにくい

## 提供するもの

KDU.map は、利用者向けの地図ビューワーと、地図データを管理するためのエディターを段階的に提供します。

最初のリリースでは、以下の機能を提供します。

- キャンパス全体の地図表示
- イベント会場や企画情報の表示

将来的には、以下の機能を提供します。

- 建物、教室、施設の検索
- 建物内部の地図表示
- 目的地までの経路案内
- バリアフリー経路や設備情報の表示

また、管理者・編集者向けのエディターとして、以下の機能を提供します。

- 建物、部屋、通路などのポリゴン編集
- 施設名、教室番号、説明文などの詳細情報の入力
- 階層構造や出入口、階段、エレベーターなどの情報管理
- 地図データを継続的に更新するための仕組み

まずは利用者向けの基本的なビューワーを開発し、その後、地図データを継続的に更新するためのエディターを整備します。

## 開発フェーズ

KDU.map は、段階的に開発します。

- Phase 1: 生駒祭向けに、キャンパス地図とイベント会場情報を表示する
- Phase 2: 建物、教室、施設の検索を実現する
- Phase 3: 建物内部の地図表示と詳細情報の管理を実現する
- Phase 4: バリアフリー情報の表示を実現する
- Phase 5: 目的地までの経路案内を実現する
- Phase 6: 他大学や他施設でも利用できるように汎用化する

## 大切にする方針

KDU.map では、以下の方針を大切にします。

- 利用者が迷わず使えること
- 正確で更新しやすい地図データを保つこと
- スマートフォンでも快適に利用できること
- 外部システムと疎結合に連携できること
- リーダブルで管理しやすいコードを書くこと
- 手作業に依存しすぎず、自動化できる部分は自動化すること

文化祭システムなどの外部システムを KDU.map に吸収するのではなく、KDU.map は独立して動作できる状態を保ちます。外部システムとは REST API などの明確な窓口を通じて連携し、必要に応じて接続できる設計を目指します。

## やらないこと

KDU.map は、以下のことを目的としません。

- 大学公式システムを置き換えること
- 授業情報、成績情報、個人情報などを扱うこと
- 利用者の現在地を継続的に追跡すること
- 文化祭システムなどの外部システムそのものを内包すること
- すべての施設情報を最初から完全に網羅すること

## 将来的な展望

まずは近畿大学東大阪キャンパスで実用できる品質を目指します。その過程で得られた地図データの管理方法、建物内部の表現方法、検索や経路案内の仕組みをもとに、将来的にはどの大学でも利用可能なインドア WebGIS アプリケーションへ発展させることを目指します。
